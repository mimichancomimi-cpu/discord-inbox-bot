## 📝 主な機能

### 1. アカウント設計ファイルから自動でプロンプト生成（⭐改善）

- アカウント設計ファイル（.md）を読み込む
- プロンプトテンプレートと組み合わせて最適なプロンプトを生成
- **新機能**: 世界観・理念、投稿テンプレート、チェックリストを自動的にプロンプトに含める
- **新機能**: 過去投稿のパフォーマンス分析（エンゲージメント率が高い投稿を優先的に参考）

### 2. AIで投稿生成（2つの方法）

#### 方法A: OpenAI APIを使用（自動）

```bash
# config.jsonにAPIキーを設定するか、--api-keyで指定
python3 投稿スケジュール/投稿作成とスケジュール.py "アカウントファイル.md" --api-key YOUR_API_KEY
```

#### 方法B: CursorのAIチャットを使用（手動）

1. プロンプトのみを表示
```bash
python3 投稿スケジュール/投稿作成とスケジュール.py "アカウントファイル.md" --prompt-only
```

2. 表示されたプロンプトをCursorのAIチャットに貼り付け
3. 生成された投稿をコピー
4. 投稿を指定してスプレッドシートに転記
```bash
python3 投稿スケジュール/投稿作成とスケジュール.py "アカウントファイル.md" --posts "生成された投稿"
```

### 3. スプレッドシートへの自動転記

- 投稿内容と日時を同時に設定
- 1日2回（朝と夜）で自動予約
- config.jsonのaccount_mappingsから自動的にスプレッドシートIDを取得

### 4. 投稿品質チェック機能（⭐新機能）

- 生成された投稿がアカウント設計に合っているかを自動チェック
- トーン&マナー、キーワード含有、投稿構造、文字数、絵文字の使用をチェック
- 品質スコアを表示（60%未満は警告、80%以上は良好）

### 5. 複数候補生成機能（⭐新機能）

- 1投稿あたり複数の候補を生成可能（`--candidates`オプション）
- 品質スコアが最も高い候補を自動選択
- より高品質な投稿を生成できる

## ⚙️ オプション一覧

### 基本オプション

- `--list`: 利用可能なアカウント一覧を表示
- `--count N`: 生成する投稿数（デフォルト: 1）
- `--theme "テーマ"`: 投稿のテーマを指定
- `--post-type {短文|中文|長文}`: 投稿タイプを指定
- `--emoji {使用する|最小限|使用しない}`: 絵文字の使用方針（デフォルト: 最小限）

### 投稿指定オプション

- `--posts "投稿1" "投稿2" ...`: 投稿内容を直接指定（AI生成をスキップ）

### AI生成オプション

- `--api-key KEY`: OpenAI APIキーを指定
- `--model MODEL`: 使用するAIモデル（デフォルト: gpt-4o-mini）
- `--prompt-only`: プロンプトのみを表示（投稿生成は行わない）
- `--candidates N`: 1投稿あたりの候補数（デフォルト: 1、複数候補から最適なものを選択）
- `--check-quality`: 生成された投稿の品質をチェック（デフォルト: 有効）
- `--no-check-quality`: 品質チェックをスキップ
- `--check-duplicate`: 過去投稿との重複をチェック（デフォルト: 有効）
- `--no-check-duplicate`: 重複チェックをスキップ

### スケジュールオプション

- `--morning-hour H`: 朝の投稿時間（時、デフォルト: 9）
- `--morning-minute M`: 朝の投稿時間（分、デフォルト: 0）
- `--evening-hour H`: 夜の投稿時間（時、デフォルト: 21）
- `--evening-minute M`: 夜の投稿時間（分、デフォルト: 0）
- `--start-date YYYY-MM-DD`: 開始日（デフォルト: 今日）
- `--no-schedule`: スプレッドシートへの転記をスキップ

## 🎯 投稿精度向上のための機能

### 1. プロンプト生成の改善

- **世界観・理念の自動抽出**: アカウント設計ファイルから世界観と理念を自動抽出し、プロンプトに含めます
- **投稿テンプレートの活用**: 成功パターンのテンプレートをプロンプトに含めます
- **チェックリストの自動追加**: 投稿作成時のチェックリストをプロンプトに含め、AIが自己チェックできるようにします
- **パフォーマンス分析**: 過去投稿のエンゲージメント率を分析し、成功パターンを優先的に参考にします
- **パフォーマンス分析の自動反映**: 実際の投稿パフォーマンスデータから成功パターンを自動抽出し、プロンプトに反映します

### 2. パフォーマンス分析機能（⭐新機能）

過去投稿のパフォーマンスデータを分析して、成功パターンを自動抽出します：

```bash
# パフォーマンス分析を実行
python3 投稿スケジュール/パフォーマンス分析.py
```

**分析内容:**
- 成功投稿と失敗投稿の比較
- 最適な文字数の分析
- 問いかけ、CTA、絵文字の効果測定
- エンゲージメント率の計算

**自動反映:**
- 分析結果は自動的にプロンプトに含まれます
- 新しい投稿生成時に成功パターンが活用されます

### 3. 投稿品質チェック

生成された投稿を以下の観点で自動チェックします：

- **トーン&マナー**: アカウントの語り口に合っているか
- **キーワード含有**: 重要なキーワードが含まれているか
- **投稿構造**: 問いかけやCTAが含まれているか
- **文字数**: 適切な長さか
- **絵文字**: 使用方針に合っているか

### 4. 重複チェック機能（⭐新機能）

生成された投稿が過去の投稿と重複していないかを自動チェックします：

- **類似度チェック**: 過去投稿との類似度を計算（70%以上で警告）
- **キーワード重複チェック**: 同じ単語が多く含まれていないか
- **自動警告**: 重複が検出された場合は警告を表示

### 5. 複数候補生成

- `--candidates 3` のように指定すると、1投稿あたり3個の候補を生成
- 品質スコアが最も高い候補を自動選択
- より高品質な投稿を生成できる

### 使用例

```bash
# 品質チェック付きで5個の投稿を生成（各投稿について3個の候補から最適なものを選択）
python3 投稿スケジュール/投稿作成とスケジュール.py \
  "５０アカウント運用/1.ぽんこつ母ちゃん.md" \
  --count 5 \
  --candidates 3 \
  --check-quality

# 品質チェックをスキップして高速に生成
python3 投稿スケジュール/投稿作成とスケジュール.py \
  "５０アカウント運用/1.ぽんこつ母ちゃん.md" \
  --count 5 \
  --no-check-quality
```
