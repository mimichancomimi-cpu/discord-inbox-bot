# スプレッドシート連携手順（詳細版）

このガイドでは、PythonスクリプトからGoogleスプレッドシートに書き込むための認証設定を行います。

## 📋 前提条件

- Googleアカウントを持っていること
- スプレッドシートへのアクセス権限があること

## 🔧 ステップ1: Google Cloud Consoleでプロジェクトを作成

### 1-1. Google Cloud Consoleにアクセス

1. ブラウザで [Google Cloud Console](https://console.cloud.google.com/) を開く
2. Googleアカウントでログイン

### 1-2. プロジェクトを作成

1. 画面上部のプロジェクト選択ドロップダウンをクリック
2. 「新しいプロジェクト」をクリック
3. プロジェクト名を入力（例: `threads-poster`）
4. 「作成」をクリック
5. プロジェクトが作成されるまで数秒待つ

## 🔧 ステップ2: APIを有効化

### 2-1. Google Sheets APIを有効化

1. 左側のメニューから「APIとサービス」→「ライブラリ」を選択
2. 検索ボックスに「Google Sheets API」と入力
3. 「Google Sheets API」をクリック
4. 「有効にする」ボタンをクリック
5. 有効化が完了するまで数秒待つ

### 2-2. Google Drive APIを有効化

1. 同じ「ライブラリ」ページで「Google Drive API」を検索
2. 「Google Drive API」をクリック
3. 「有効にする」ボタンをクリック
4. 有効化が完了するまで数秒待つ

## 🔧 ステップ3: サービスアカウントを作成

### 3-1. サービスアカウントを作成

1. 左側のメニューから「APIとサービス」→「認証情報」を選択
2. 画面上部の「+ 認証情報を作成」をクリック
3. 「サービスアカウント」を選択

### 3-2. サービスアカウントの詳細を入力

1. **サービスアカウント名**: 任意の名前を入力（例: `threads-poster-service`）
2. **サービスアカウントID**: 自動生成されます（変更可能）
3. **説明**: 任意（例: `Threads投稿用サービスアカウント`）
4. 「作成して続行」をクリック

### 3-3. ロールの設定（オプション）

1. 「このサービスアカウントにロールを付与」は**スキップ可能**です
2. 「続行」をクリック
3. 「完了」をクリック

## 🔧 ステップ4: JSONキーをダウンロード

### 4-1. サービスアカウントのキーを作成

1. 作成したサービスアカウントをクリック（一覧に表示されます）
2. 「キー」タブを選択
3. 「キーを追加」→「新しいキーを作成」をクリック
4. **キーのタイプ**: 「JSON」を選択
5. 「作成」をクリック

### 4-2. JSONファイルを保存

1. JSONファイルが自動的にダウンロードされます
2. ファイル名は `プロジェクト名-xxxxx.json` のような形式です
3. このファイルを安全な場所に保存してください

**⚠️ 重要**: このJSONファイルは機密情報です。他人に共有しないでください。

## 🔧 ステップ5: 認証情報ファイルを配置

### 5-1. ファイル名を変更

1. ダウンロードしたJSONファイルを `credentials.json` にリネーム

### 5-2. ファイルを配置

1. `/Users/sasaki/Documents/Obsidian Vault/` フォルダに `credentials.json` を配置

**配置後のパス例:**
```

```

## 🔧 ステップ6: スプレッドシートに権限を付与

### 6-1. サービスアカウントのメールアドレスを確認

1. ダウンロードしたJSONファイル（`credentials.json`）を開く
2. `client_email` の値を確認（例: `threads-poster-service@プロジェクト名.iam.gserviceaccount.com`）

### 6-2. 各スプレッドシートに権限を付与

**重要**: 使用する**すべてのスプレッドシート**に対して権限を付与する必要があります。

1. スプレッドシートを開く
2. 右上の「共有」ボタン（または「共有」アイコン）をクリック
3. 「ユーザーやグループを追加」欄に、サービスアカウントのメールアドレスを入力
4. 権限を「編集者」に設定
5. 「送信」をクリック

## ❌ よくあるエラーと対処法

### エラー1: "認証情報ファイルが見つかりません"

**原因**: `credentials.json` のパスが間違っている

**対処法**:
- `config.json` の `credentials_path` を確認
- ファイルが実際に存在するか確認

### エラー2: "Permission denied" または "アクセス権限がありません"

**原因**: スプレッドシートにサービスアカウントの権限が付与されていない

**対処法**:
- 各スプレッドシートにサービスアカウントのメールアドレスを「編集者」として追加
- サービスアカウントのメールアドレスを確認（`credentials.json` の `client_email`）

### エラー3: "API not enabled"

**原因**: Google Sheets APIまたはGoogle Drive APIが有効になっていない

**対処法**:
- Google Cloud ConsoleでAPIが有効になっているか確認
- 必要に応じて再度有効化

### エラー4: "Worksheet not found"

**原因**: 指定したシート名が存在しない

**対処法**:
- `--list-sheets` オプションで利用可能なシートを確認
- 正しいシート名を指定
